# Commit2Zen

Git 提交记录管理与禅道任务自动化工具。自动收集 Git 提交、按分支汇总，并一键在禅道中创建任务。

## 功能特性

- **项目扫描**: 选择文件夹自动扫描 Git 仓库
- **多禅道账号**: 支持配置多个禅道账号，执行时自由选择
- **提交预览**: 预览提交记录，按分支自动分组汇总
- **一键执行**: 为每个分支自动在禅道创建任务
- **历史报告**: 查看历次执行报告

## 快速开始

### 方式一：使用安装包（推荐）

1. 前往 [Releases](https://github.com/benbenlast/Commit2Zen/releases) 下载最新安装包
2. 安装后打开应用

### 方式二：从源码构建

**前置要求**

- [Rust](https://rustup.rs/)
- [Node.js](https://nodejs.org/) 18+

```bash
# 安装依赖
npm install

# 开发模式
npm run tauri dev

# 构建安装包
npm run tauri build
```

构建产物位于 `src-tauri/target/release/bundle/`

## 使用说明

### 第一步：配置禅道账号

1. 打开应用，进入「配置」页面
2. 点击「+ 新增」添加禅道账号
3. 填写显示名称、禅道地址、账号、密码、默认指派人、任务类型
4. 点击「测试连接」验证配置是否正确
5. 可添加多个禅道账号

### 第二步：选择 Git 项目

1. 进入「执行」页面
2. 点击「选择扫描目录」选择包含 Git 仓库的文件夹
3. 等待扫描完成后，从列表中选择目标项目

### 第三步：配置禅道并预览

1. 选择一个已保存的禅道账号
2. 系统自动连接禅道并获取项目列表
3. 选择目标项目
4. 点击「下一步：预览」查看提交和分支汇总

### 第四步：执行创建任务

1. 预览页面展示所有分支的提交记录和统计信息
2. 点击「创建禅道任务」为每个分支在禅道中创建任务
3. 执行完成后展示每个分支的任务创建结果和链接

## 配置选项

| 字段 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| 禅道地址 | ✅ | 禅道服务器地址 | - |
| 禅道账号 | ✅ | 登录账号 | - |
| 禅道密码 | ✅ | 登录密码 | - |
| 默认指派人 | ✅ | 任务指派给谁 | - |
| 任务类型 | ❌ | dev/design/qa/research | dev |
| 最大提交数 | ❌ | 每次最多读取多少条 | 100 |
| 分支过滤 | ❌ | 正则表达式过滤分支 | .* |

## 错误处理

- 网络请求失败会自动重试（最多 3 次，指数退避）
- 单个分支任务创建失败不影响其他分支
- 所有错误记录在报告文件中

## 安全注意

- 配置数据保存在本地 `~/.config/Commit2Zen/config.json`
- 包含敏感信息，已加入 `.gitignore`
- 不要在版本控制中提交配置文件
